# Audit — Auth Google OIDC (apps/api) — 2026-06-19

- **Auteur** : agent REVUE-SÉCURITÉ (Opus), boucle multi-agents.
- **Périmètre** : `apps/api/src/{app,auth/*,config/env,store/*,logging/redact,server}.ts` + tests.
- **Référence** : `security-threat-model.md` (18 must-have), `feature-auth-api.md` (AUTH-D-1..8).

## Findings & état

| ID | Sév. | Description | État |
|----|------|-------------|------|
| SEC-C1/C2/E1 | 🔴 Critique | `state` ni à usage unique ni consommé serveur → rejeu du callback / login-CSRF (must-have #3). | ✅ **Corrigé** : `TxStore` serveur, cookie ne porte qu'un id, `consume` atomique (getAndDelete) au callback ; cookie `oidc_tx` effacé dans tous les cas. **Test de rejeu ajouté** (2ᵉ callback → 400 `tx_expired`). |
| SEC-E2 | 🟠 Élevé | Ancienne session non détruite au login (must-have #6). | ✅ **Corrigé** : `rotateSession` supprime l'ancien `sid` avant d'en créer un nouveau. |
| SEC-M1 | 🟡 Moyen | Logger off → pas d'audit, `redact`/`maskEmail` morts (must-have #16/17). | ✅ **Corrigé** : pino activé (silencieux en test), **logs d'audit** login/logout/account_deleted/login_failed, email masqué via `maskEmail`. |
| SEC-M2 | 🟡 Moyen | `trustProxy:true` inconditionnel → spoof IP, rate-limit contournable (must-have #12). | ✅ **Corrigé** : `TRUST_PROXY_HOPS` (défaut 0) configurable. |
| SEC-M3 | 🟡 Moyen | `JSON.parse` du tx non validé. | ✅ **Résolu par conception** : le cookie ne contient plus de JSON (juste un id), plus de parse. |
| SEC-E3 | 🟡 Moyen | Pas d'idle timeout (que TTL absolu). | ⏳ **Backlog** (non bloquant). |
| SEC-F1 | 🟢 Faible | CSP helmet par défaut (pas « stricte »). | ⏳ **Backlog** (durcir quand le front sera servi par l'API). |
| SEC-C2 (résiduel) | 🟡 | Liaison du state à un secret navigateur distinct (au-delà de l'usage unique). | ⏳ **Backlog** : à renforcer lors du passage Prisma. |

## Must-have OIDC validés (tests à l'appui)
PKCE S256 ✅ · validation `id_token` complète **RS256 épinglé** (pas d'`alg:none`/HS) ✅ · `state` aléatoire + **usage unique** ✅ · mapping par `sub` + `email_verified` ✅ · cookies httpOnly/Secure/SameSite=Lax ✅ · CSRF double-submit temps constant ✅ · suppression de compte RGPD ✅ · audit + masquage PII ✅.

## Reste (backlog, non bloquant ce jalon)
- **Persistance Prisma/Postgres** (AUTH-D-8) : remplacer les stores in-memory + migrations + reconduire `getAndDelete` atomique du state et l'invalidation de session en base. **Avant tout déploiement.**
- Idle timeout (SEC-E3), CSP stricte (SEC-F1), liaison renforcée du state (SEC-C2 résiduel).
- CI `pnpm audit` (#18).

## Verdict
Cœur OIDC solide et testé ; les 3 manques bloquants (state usage-unique, rotation session, audit/trustProxy) sont **corrigés**. Auth déployable **après** bascule Prisma (persistance) qui doit reconduire ces invariants.
